# Design register

## Visual direction
- Product UI first
- Clean, calm, and trustworthy
- Slightly warm neutrals with a blue accent for primary actions
- Dense enough for scanning, but never cramped

## Color strategy
- Strategy: Restrained with a clear blue accent
- Backgrounds: soft slate or neutral off-white surfaces
- Primary: blue for actions, active states, and links
- Success: green for confirmations and stable status
- Warning and error: standard semantic tones, used sparingly

## Typography
- Use a single sans family across the product
- Prefer system-safe sans stacks or Inter-style rendering
- Strong hierarchy on headings, compact body text, readable labels
- Avoid display typography in controls, forms, or data surfaces

## Layout
- Standard app patterns: top bar, content area, filter sidebar, cards, panels
- Predictable grids and consistent spacing
- Use whitespace to separate groups, not decoration
- Keep pages task-focused and responsive by structure

## Component style
- Rounded but not bubbly
- Buttons should share a consistent shape vocabulary
- Form controls should feel native and clear
- Cards should be used only when they improve scanability
- Empty states should teach the next action

## Motion
- Short, state-driven transitions only
- No decorative choreography
- Subtle hover, focus, and reveal behavior
- Use motion to confirm interaction, not to entertain

## Spacing rhythm
- Spacious on the homepage
- Denser on search and listing surfaces
- Keep filter groups visually separated
- Maintain comfortable tap targets on mobile

## Icon style
- Use simple emoji or plain glyph icons consistently
- Match icon meaning to the action or amenity
- Keep icon usage functional, not ornamental

## UI rules
- Primary CTA should always be obvious
- Search and filters should remain easy to scan
- The map should support discovery, not overpower the listing list
- Listing cards should prioritize price, location, area, and key amenities
- Posting flow should feel calm, guided, and high confidence
